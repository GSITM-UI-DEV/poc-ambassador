# 06. 개발 진행 계획

## 작업 순서

| 단계 | 작업 | 산출물 |
|---|---|---|
| M0 | 프로젝트 셋업 | Vite+React+TS, Tailwind v4, react-router, 폰트(Pretendard/Playfair Display), 디자인 토큰(`03-design.md` → Tailwind theme) |
| M1 | 공통 기반 | 목업 데이터(`04-data-model.md`), zustand store(auth/booking), 공용 컴포넌트(Button·Badge·Card·PriceBlock), PcLayout/MobileLayout |
| M2 | PC 메인 | Figma 452:309 재현 — 헤더/히어로+검색위젯/추천혜택/Promotions/멤버십/하루/티어/푸터 |
| M3 | 모바일 플로우 7화면 | 홈 → 검색 → 상세 → 옵션 → 혜택 → 결제 → 완료 (예약 상태 연결 포함) |
| M4 | PC 여정 페이지 | 로그인 → 검색 결과 → 호텔 상세 → 예약 3-step → 완료 |
| M5 | 마이페이지 + 스페셜 오퍼 | TanStack Table 예약 내역, 멤버십 대시보드, 오퍼 목록 |
| M6 | 마감 | `/guide` 페이지 URL 가이드(단순 테이블 인덱스), 여정 전체 시연 점검(PC/모바일), 빈 상태·리다이렉트 처리, 스타일 폴리시 |

> M3(모바일)를 M4(PC 여정)보다 먼저: 모바일이 Figma 필수 범위이고, 예약 플로우 로직(가격 계산·상태)을 모바일에서 완성한 뒤 PC에서 재사용하는 편이 효율적.

## 검증 방법

- 각 마일스톤 종료 시 dev 서버에서 Playwright로 스크린샷 캡처 → Figma 원본과 비교 (특히 M2, M3)
- 예약 플로우 가격 계산은 Figma 수치로 검증: 345,000 + 35,000 - 32,000 - 10,000 = 338,000원
- 모바일 화면은 뷰포트 390px 기준 확인

## 결정 사항 기록

- 반응형이 아닌 **URL 분리형** (`/` vs `/mobile`) — 사용자 지정
- 백엔드 없음, 전부 목업 — 시연용 프로토타입
- Figma에 없는 화면(PC 로그인·검색·상세·마이페이지 등)은 `03-design.md` 토큰 기준으로 자체 설계

## 리스크 / 메모

- Figma 디자인이 작업 중이므로 변경 가능 → 토큰만 갱신하면 전체 반영되도록 하드코딩 컬러 금지
- 이미지가 전부 실사진 → Unsplash placeholder로 대체, 추후 실제 에셋 교체 용이하도록 데이터에 URL 필드로 관리
- 모바일 Figma의 빈 프레임 2개(511:80, 511:83)는 미완성 화면으로 판단, 범위 제외
