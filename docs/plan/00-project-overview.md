# 00. 프로젝트 개요

## 목적

엠배서더호텔 그룹 홈페이지 재구축 사업(RFP) 제안을 위한 **클릭 가능한 프로토타입** 제작.
디자이너 작업이 진행 중인 상황에서, Figma의 톤&테마를 유지한 채 핵심 사용자 여정(탐색 → 예약 → 결제 → 멤버십)을 시연할 수 있는 수준을 목표로 한다.

## 핵심 설계 방향 (제안서 3축 반영)

1. **가치제안** — 정가/회원가/할인율/절감액 동시 노출, 등급 배지(GOLD 15%), 공식 채널 전용 혜택
2. **고객경험** — 목적 기반 탐색(기념일·가족·미식 등), 객실+다이닝 원스톱 예약, 단계 최소화
3. **고객관계** — 멤버십 대시보드(포인트·등급 게이지·소멸 예정), 생애주기 추천

## 범위

- **PC 웹** (`/`) + **모바일 웹** (`/mobile`) — 라우팅으로 디바이스 구분 (반응형 아님, 각각 별도 레이아웃)
- 프론트엔드 전용 프로토타입: 백엔드 없음, 목업 데이터(JSON/TS 상수) + 클라이언트 상태로 시연
- 모바일은 **Figma 모바일 플로우 7개 화면 전부 구현** (필수)
- PC는 로그인·메인·예약하기 필수 + 여정 완성에 필요한 페이지 추가 (상세는 `02-sitemap-routing.md`)

## 제외 범위 (프로토타입에서 안 만드는 것)

- 실제 인증/결제/PG 연동 (화면 시연용 가짜 처리)
- 관리자(Admin) 전체, 상품권 관리, E-Shop — RFP에 있으나 프로토 범위 밖
- 다국어, 접근성 완전 준수, 성능 최적화
